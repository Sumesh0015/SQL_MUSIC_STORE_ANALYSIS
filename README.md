# SQL Project: Music Store Data Analysis
## Project Overview

This project focuses on analyzing a Music Store database using SQL (PostgreSQL) to extract meaningful business insights related to sales performance, customer behavior, and artist popularity.

The objective of this project is to demonstrate strong SQL fundamentals, analytical thinking, and the ability to solve real-world business questions using data.

This project is well-suited for Data Analyst / Business Analyst / Decision Analyst fresher roles.

## Tools & Technologies

Database: PostgreSQL

SQL Client: pgAdmin 4

Language: SQL

Version Control: Git & GitHub

## Business Questions Solved

This project answers key business and analytical questions such as:

Who are the best-selling artists?

Which customers spend the most and on which artists?

What are the top-performing genres?

Which countries generate the highest revenue?

How does customer spending vary across regions?

What is the overall revenue and sales distribution?

## Key SQL Concepts Used

JOIN (INNER JOIN, multiple table joins)

GROUP BY & ORDER BY

Aggregate functions (SUM, COUNT, AVG)

CTEs (WITH clause)

Subqueries

Filtering with WHERE

Sorting & ranking results

Business-oriented query structuring

## Key Insights

Identified the top-selling artist based on total revenue.

Found high-value customers and their spending patterns.

Analyzed country-wise revenue contribution.

Discovered popular genres contributing most to sales.

Highlighted customer segments useful for targeted marketing.

These insights can directly support business decision-making such as promotions, inventory planning, and regional strategy.
